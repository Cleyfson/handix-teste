Avaliação Técnica: Fullstack Developer
Introdução
Esta avaliação faz parte do processo seletivo da Handix. O objetivo é avaliar a sua capacidade de
construir uma aplicação funcional de ponta a ponta, utilizando Laravel no backend e o framework da
sua preferência no frontend.
O Desafio: Sistema de Gestão de Contatos
Você deve criar um CRUD (Create, Read, Update, Delete) simples para gerenciar uma lista de contatos.
1. Requisitos do Backend (Laravel)
• API Endpoint: Criar uma API RESTful para os contatos.
• Campos do Contato: Nome, E-mail, Telefone e Observações.
• Validação: Garantir que o e-mail seja único e o nome seja obrigatório.
• Persistência: Utilizar Migrations para estruturar a base de dados.
2. Requisitos do Frontend
• Interface: Uma tela onde seja possível visualizar a listagem, adicionar novo contato, editar e excluir.
• Funcionalidade de Busca: Implementar um campo de busca para filtrar os contatos na listagem.
• Integração: Consumir a API criada no Laravel utilizando Axios ou Fetch API.
• Tecnologia: Você tem liberdade para utilizar Vue.js, React ou qualquer outro framework frontend de
sua preferência.
• Componentização: Estruturar o código de forma organizada.
Diferenciais (Opcional)
• Uso de Docker (Laravel Sail).
• Boas práticas de UI/UX simples.
Instruções de Entrega
1. O código deve ser hospedado em um repositório Git (GitHub, GitLab ou Bitbucket).
2. O repositório deve ser público ou você deve conceder acesso ao avaliador.
3. Inclua um arquivo README.md com instruções de execução e decisões técnicas relevantes.
Critérios de Avaliação
• Organização do Código e boas práticas.
• Domínio da Stack (Laravel e ferramenta frontend escolhida).
• Tratamento de Erros e Lógica de programação.
